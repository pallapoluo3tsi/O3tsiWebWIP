      FTS Wind Tunnelâ„¢

      Main Components

      Wind Tunnel Console
      & Central Database

      Used to control the test setup, test execution and test monitoring of the test applications under test within the Wind Tunnel Environment. It stores all required information to support the testing process.

      Wind Tunnel
      Control Channel

      The mechanism used to communicate and transfer information between the Wind Tunnel Console and the Wind Tunnel Environment

      Wind Tunnel
      Test Environment

      Where the testing of the applications under test take place.

      Sub Components

      • Test Environment Configurations

        Configurations to set up and launch the applications under test.

      • Test Data Configurations

        Configurations to connect and synchronize data for the applications under test.

      • Test Packs

        Test scenarios required to perform functional, regression and perfomance testing for the applications under test.

      • Test Automation

        Test automation required to perfom functional, regression and performance testing for the applications under test.

      • Applications Under Test

        Actual applications or simulators simulating applications under test.

      • Patch Panel Component

        This component executes the setup and connection of the applications under test leveraging switches and the Logical Control Channel.

      • Logical Connection Channel

        The logical channel used to connect the applications under test.

      • Agent-Based Component

        This component includes the agents required to perform and monitor the tests within and across the applications under test.

      • The outputs or test results from the application under test.
